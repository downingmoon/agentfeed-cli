# CLI Test Browser Side Effect Guard

> [!summary]
> CLI test runs could open the host browser when a publish/review handoff path reached the real OS opener. Vitest now sets a test-only browser safety flag, and `openBrowser` refuses to launch a real browser unless a test opener shim is explicitly installed.

## Root cause

- Default project config has `collection.open_review_after_upload: true`.
- Human-mode `agentfeed publish` calls the review URL handoff after upload.
- The test fixture URL `http://localhost:3001/worklogs/worklog_publish_confirmed/review` is generated by `tests/cli-share.test.ts`.
- If a test path reaches `openBrowser` without CI suppression or the fake opener shim on `PATH`, macOS `open` can launch a real browser tab/window.

## Change

- `vitest.config.ts` sets `AGENTFEED_TEST_DISABLE_REAL_BROWSER=1` for test processes.
- `src/utils/open-browser.ts` returns `false` instead of spawning `open`/`xdg-open`/`wslview` when that flag is set and no `AGENTFEED_TEST_BROWSER_LOG` fake opener is installed.
- `tests/open-browser.test.ts` locks the guard behavior while preserving explicit fake-opener tests.

## Verification

- `npm test -- --run tests/open-browser.test.ts tests/cli-share.test.ts -t "allows --yes|refuses to launch a real browser|opens the review URL after publish when project config enables it|does not auto-open"` → 4 tests passed.
- `node --check scripts/release-preflight.mjs && npm test -- --run tests/release-preflight.test.ts tests/version.test.ts` → 14 tests passed.
